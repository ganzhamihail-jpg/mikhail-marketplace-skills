# mikhail-marketplace-skills

Набор прикладных навыков для AI-агентов, которые помогают управлять бизнесом на Wildberries, Ozon и других маркетплейсах. Навыки построены в переносимом формате Agent Skills и совместимы со SkillKit, Codex и другими агентами, поддерживающими `SKILL.md`.

## Состав

| Навык | Назначение |
|---|---|
| `wb-advertising-expert` | Аудит рекламы Wildberries, ДРР, ставки, запросы и план оптимизации |
| `marketplace-finance-analyst` | Сверка отчетов, P&L, удержания, расхождения и причины изменения прибыли |
| `unit-economics` | Юнит-экономика SKU, безубыточная цена, маржа и предельный ДРР |
| `warehouse-localization` | Пополнение, локализация и перераспределение запасов между складами |
| `weekly-profit-report` | Еженедельный управленческий отчет по прибыли |
| `telegram-post-generator` | Практичные Telegram-посты для продавцов и клиентов |
| `marketplace-teacher` | Уроки, программы, упражнения и проверка знаний |
| `certificate-document-validator` | Проверка сертификатов, реестров и привязок документов к товарам |

## Структура

```text
mikhail-marketplace-skills/
├── README.md
├── LICENSE
└── skills/
    └── <skill-name>/
        ├── SKILL.md
        └── agents/
            └── openai.yaml
```

`SKILL.md` содержит обязательные метаданные `name` и `description`, а также инструкции агента. `agents/openai.yaml` добавляет отображаемое имя, короткое описание и стартовый промпт для Codex.

## Установка через SkillKit

Требуется Node.js 18 или новее.

Посмотреть найденные навыки:

```bash
npx skillkit@latest install ganzhamihail-jpg/mikhail-marketplace-skills --list
```

Установить весь набор:

```bash
npx skillkit@latest install ganzhamihail-jpg/mikhail-marketplace-skills
```

Установить один или несколько навыков:

```bash
npx skillkit@latest install ganzhamihail-jpg/mikhail-marketplace-skills \
  --skill unit-economics weekly-profit-report
```

SkillKit предложит выбрать целевой агент и каталог. Для Codex навыки обычно синхронизируются в `.codex/skills/` проекта или в настроенный каталог навыков.

## Локальная установка при разработке

Из корня репозитория:

```bash
npx skillkit@latest install . --list
npx skillkit@latest install . --skill unit-economics
```

## Использование

Навык можно вызвать явно по имени:

```text
Используй $unit-economics. Рассчитай прибыль на одну выкупленную единицу:
цена 1 990 ₽, комиссия 24%, логистика 145 ₽, себестоимость 620 ₽,
реклама 180 ₽, налог 6%. Покажи безубыточную цену.
```

Или сформулировать естественный запрос — агент выберет навык по описанию:

```text
Проверь рекламный отчет Wildberries, найди убыточные кампании и предложи
план изменения ставок на неделю.
```

## Тестовые промпты

### wb-advertising-expert

```text
Используй $wb-advertising-expert. Кампания потратила 32 000 ₽, получила
4 000 кликов, 160 заказов и 128 выкупов на 256 000 ₽. Маржинальность до
рекламы — 28%. Рассчитай метрики, оцени прибыльность и предложи действия.
```

### marketplace-finance-analyst

```text
Используй $marketplace-finance-analyst. Построй мост прибыли: выручка
850 000 ₽, возвраты 70 000 ₽, комиссия 162 000 ₽, логистика 91 000 ₽,
реклама 88 000 ₽, себестоимость 280 000 ₽. Покажи допущения.
```

### unit-economics

```text
Используй $unit-economics. Цена после скидок 2 490 ₽, комиссия 25%,
логистика 170 ₽, себестоимость 780 ₽, налог 6%, ДРР 12%.
Рассчитай прибыль, маржу и предельный ДРР.
```

### warehouse-localization

```text
Используй $warehouse-localization. Волгоград — центральный склад.
Есть остатки и продажи за 28 дней по четырем складам. Раздели результат
на пополнение Волгограда и перераспределение по остальным складам.
```

### weekly-profit-report

```text
Используй $weekly-profit-report. Подготовь отчет собственнику за 13–19 июля
и сравни с 6–12 июля. Начни с изменения прибыли и трех главных причин.
```

### telegram-post-generator

```text
Используй $telegram-post-generator. Напиши пост до 1 200 знаков для продавцов
WB о том, почему низкий ДРР не всегда означает прибыльную рекламу.
Добавь один числовой пример и спокойный CTA.
```

### marketplace-teacher

```text
Используй $marketplace-teacher. Создай 60-минутный урок для начинающего
менеджера по теме «Юнит-экономика SKU»: теория, пример, упражнение,
чек-лист проверки и ответы.
```

### certificate-document-validator

```text
Используй $certificate-document-validator. Проверь реестр из 20 SKU и 7 PDF:
найди истекшие документы, дубли и неоднозначные привязки. Сначала подготовь
сухой план загрузки, ничего не отправляй в API.
```

## Принципы безопасности и качества

- Не выдумывать отсутствующие показатели, документы или правила площадок.
- Проверять актуальные тарифы, требования и интерфейсы по первичным источникам.
- Хранить токены, ключи API и учетные данные вне таблиц, репозитория и чатов.
- Не менять исходные отчеты и шаблоны без необходимости.
- Не объединять неоднозначные SKU автоматически.
- Перед массовой загрузкой выполнять сухой запуск и сохранять журнал результата.
